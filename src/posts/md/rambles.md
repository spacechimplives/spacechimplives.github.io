When someone decides to hire a lawyer, they are not looking for information, they are looking for an outcome. And to the extent that they can afford it and it is worth the expense, they will continue hiring people until that outcome is achieved.

Similarly, when someone hires a laborer or researcher, they are doing so as a means to receive a product.  Any information is incidental, so the outcomes are either a translation from the original information to the product, or a response of "impossible". 

This is similar to a function which returns either an object with certain properties, or 'null'.  

The way to handle this in an organic context is to instantiate a new object or return an object which has already pre-defined those properties.
